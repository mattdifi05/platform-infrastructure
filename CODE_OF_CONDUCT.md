# Code of Conduct

## Our pledge

We aim to keep this project professional, respectful and focused on useful infrastructure work.

## Expected behavior

- Be respectful and constructive.
- Keep technical criticism specific and actionable.
- Assume maintainers need clear reproduction steps and evidence.
- Avoid sharing secrets, credentials or private customer data.

## Unacceptable behavior

- Harassment or personal attacks.
- Publishing private credentials or sensitive operational data.
- Repeated off-topic disruption.
- Misrepresenting test, security or production readiness status.

## Enforcement

Maintainers may edit, close or remove issues and pull requests that violate these expectations.
