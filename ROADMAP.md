# Roadmap

## Now

- Maintain the Docker-first local and VPS foundation.
- Keep docs white-label and maintainer-readable.
- Keep release evidence, backup/restore and go/no-go gates green.
- Preserve safe defaults for admin and internal services.

## Next

- Improve project manifest schema validation.
- Add clearer examples for external app repositories.
- Expand provider evidence examples.
- Improve Docs UI search and grouping.
- Add markdown/link checks to CI.

## Later

- Optional HA deployment profile.
- More provider adapters.
- Richer policy-as-code admission checks.
- External secrets manager integrations.

## Enterprise / Future

- Multi-node reference architecture.
- SSO/MFA policy packs.
- Formal compliance evidence exports.
- Production-grade tenant separation model.

## Out of scope

- Application source hosting inside this repository.
- Provider-specific branding.
- Public exposure of database/admin tools without an access layer.
- Replacing full platform engineering review with a single script.
